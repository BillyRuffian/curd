Clear Personally Identifiable Information On Navigation

This story describes the rules for clearing personally identifiable information
from frontend screens when a user navigates through the identity screens

Scenario: AC-01a User navigates backwards using the back link and passport number is hidden

If a user captures their passport number and then continues the applicant portal login process
The passport number will be hidden from the frontend view if they navigate backwards using the "back" link

And
I have accessed the "Do you have your National Insurance number?" page
When
I click the "Back" link
Then
I am redirected to the "Enter your UK passport number" page
And
The passport number field is blank

Scenario: AC-01b User navigates backwards using the back button in their browser and passport number is hidden

If a user captures their passport number and then continues the applicant portal login process
The passport number will be hidden from the frontend view if they navigate backwards using the browser "back" button

And
I have accessed the "Do you have your National Insurance number?" page
When
I navigate backwards in the browser 1 times
Then
I am redirected to the "Enter your UK passport number" page
And
The passport number field is blank

Scenario: AC-01c User can still log into the applicant portal with a hidden passport number after using the back link

A user can still log into the applicant portal if they have navigated backwards to the passport-number capture screen by using the "back" link
and have thus caused the passport number data to be hidden in the frontend view,
but they then navigate past the passport number screen using the browser's back button
and attempt to log into the applicant portal as normal

And
I have accessed the "Do you have your National Insurance number?" page
And
I click the "Back" link
And
I am redirected to the "Enter your UK passport number" page
And
I navigate backwards in the browser 1 times
And
I enter a valid National Insurance number
When
I proceed
Then
I am redirected to the "Your provisional licence application" page

A user can still log into the applicant portal if they have navigated backwards to the passport-number capture screen by using the browser "back" button

Scenario: AC-03a User navigates forwards after using the back link and passport number is hidden

And
I have accessed the "Do you have your National Insurance number?" page
And
I click the "Back" link 2 times

I click on Continue
Then
I am redirected to the "Enter your UK passport number" page
And
The passport number field is blank
